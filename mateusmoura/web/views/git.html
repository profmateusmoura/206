Git e GitHub

Este guia é para estudantes que estão começando a aprender Git e GitHub, com integração ao Visual Studio Code (VSCode). Os comandos estão organizados de forma sequencial para facilitar o uso.

1. Configurando o Usuário no Git

Quando o Usuário Não Está Configurado

Se você ainda não configurou seu usuário no Git, execute os comandos abaixo para definir seu nome e e-mail globalmente. Essas informações serão associadas aos seus commits.

Definindo Nome e E-mail

git config --global user.name "Seu Nome"
git config --global user.email "seuemail@exemplo.com"

Para confirmar as configurações, use:

git config --global --list

2. Primeiro Uso com VSCode

Inicializando e Configurando o Repositório

Para começar, configure seu repositório local usando o VSCode:

Inicializar o Repositório

git init

Adicionar Arquivos

git add .

Commit Inicial

git commit -m "Commit inicial"

Conectar ao Repositório Remoto e Enviar

git remote add origin https://github.com/seu-usuario/seu-repositorio.git
git push origin main

3. Trabalhando com Commits

Preparando e Registrando Suas Alterações

Após modificar seu código, utilize os comandos abaixo, diretamente pelo VSCode:

Verificar o Status

git status

Adicionar Alterações

git add .

Criar um Commit

git commit -m "Descrição das alterações"

Visualizar Histórico de Commits

git log

4. Atualizando os Commits e Sincronizando com GitHub

Sincronizando com o Repositório Remoto

Utilize estes comandos para manter seu repositório local atualizado:

Puxar Atualizações (Pull)

git pull origin main

Enviar Novos Commits (Push)

git push origin main

5. Trabalhando com Branches e Merge

Fluxo de Desenvolvimento com Branches

Para desenvolver novas funcionalidades sem afetar o código principal:

Criar uma Nova Branch

git checkout -b nova-branch

Realizar o Merge e Resolver Conflitos

Após finalizar as alterações na branch, mescle-a com a branch principal:

git checkout main
git merge nova-branch
git add .
git commit -m "Merge e resolução de conflitos"

Histórico Linear com Rebase

git rebase main

6. Recursos Avançados e Comandos Úteis

Outros Comandos Importantes

Staging Parcial

git add -p

Visualizar Diferenças

git diff HEAD

Gerenciar Branches

git branch nome-da-branch

Ignorando Arquivos com .gitignore

.vscode/
dist/
build/

Desfazendo Mudanças

Resetar o Último Commit

git reset --soft HEAD~1

Revertendo um Commit

git revert [commit]

7. Sincronizando com GitHub

Configurando o Repositório Remoto

Após configurar seu nome e e-mail com o Git (consulte a Seção 1, se necessário), conecte seu repositório local ao repositório remoto no GitHub:

git remote add origin https://github.com/seu-usuario/seu-repositorio.git

Enviando seus Commits ao GitHub

Após realizar commits locais, sincronize seu repositório com o GitHub utilizando o seguinte comando:

git push -u origin main

Atualizando seu Repositório Local

Para buscar as últimas alterações do repositório remoto:

git pull origin main
